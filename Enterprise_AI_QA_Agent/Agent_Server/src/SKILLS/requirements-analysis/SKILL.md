# Requirements Analysis

Use this skill when the user provides product requirements, feature descriptions, acceptance criteria, or ambiguous QA goals.

Focus on:
- Business goal and user journey
- In-scope and out-of-scope behavior
- Acceptance criteria and observable outcomes
- Test data, environment, and access assumptions
- Risks, dependencies, and open questions
