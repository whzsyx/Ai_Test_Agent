## 4. 发现的问题

| 优先级 | 问题 | 证据 |
|---|---|---|
| P1 | `performance-engine-runner` 在传入 plan 且 plan 缺 `engine` 时忽略顶层 `engine=jmeter`，实际返回 k6 | 直接 handler：`perf_engine_runner_jmeter_with_plan_missing_engine` 输出 `engine=k6/selected_image_key=perf_k6_default` |
| P1 | 附件补文档黑盒续跑失败，未从会话附件 `text_excerpt` 生成 `content_base64` | BB-005：`api-docs-ingest import_attachment` 报 `import_attachment 需要 content_base64 参数` |
| P2 | `http-probe-runner` 直接探测 mock runner 返回的 `127.0.0.1` 地址失败 | Docker 与 BB-001 初跑均出现 `status_code=0`；改为 `host.docker.internal` 成功 |
| P2 | 黑盒 JMeter 链路仍偏“容器生命周期验证”，未完成真实 JMeter load 报告 | BB-002 选择 JMeter 与容器 start/stop 通过，但未输出完整 JMeter load metrics |
| P3 | approval 后 tool-jobs 中保留 `waiting_approval` 记录，容易干扰自动判定 | 多个黑盒会话中同一工具存在 waiting_approval 与 completed 两条记录 |
| P3 | observability 指标与 optional 工具未实现 | OBS/OPT 矩阵 |
